# Observability standards (project-scoped)

This repo ships pino structured logging, OpenTelemetry traces (OTLP/HTTP),
Prometheus metrics on `:9464`, and Terminus health on `/health`. These rules
keep them honest and useful in production.

## Logging

1. **Use `Logger` from `@nestjs/common` or `nestjs-pino`** — never `console.log`
   in production code. The shipped `KafkaDlqFilter` and
   `SchemaValidationExceptionFilter` have stray `console.log` calls; remove
   them when touching those files.
2. **Required fields on every Kafka handler log line** (the **traceability
   quartet**):
   - `topic` — `KafkaContext.getTopic()`
   - `partition` — `KafkaContext.getPartition()`
   - `offset` — `KafkaContext.getMessage().offset`
   - `eventId` — from the decoded payload
3. **Producer log lines should include**:
   - `topic`
   - `schemaId` (read from the SR-framed buffer: `buf.readUInt32BE(1)`)
   - `bytes` (encoded payload size)
   - `eventId`
   - The `AvroSerializer` already logs these; reuse the format if you write a
     bespoke producer path.
4. **Use single-line `key=value` separators** in log message bodies (matches
   existing format). Don't embed multi-line JSON in log messages — pino's
   structured fields are the place for that.
5. **Log at `WARN` for retryable/expected errors** (handler retry attempts,
   schema validation rejections). `ERROR` is reserved for things that should
   wake somebody up.
6. **Don't log full payloads at INFO+** — they may contain PII. Log
   `eventId`, business identifiers (`userId`, `orderId`), and shapes
   (`items.length`), not raw user data.

## OpenTelemetry

7. **`import './observability/tracing'` MUST be the first import in
   `src/main.ts`.** OTel auto-instrumentation works by patching modules at
   require-time — if `kafkajs` is loaded before the SDK, traces silently miss
   the Kafka spans.
8. **Service name comes from `OTEL_SERVICE_NAME`** — keep it stable across
   environments so spans aggregate correctly in Jaeger / Honeycomb / etc.
9. **Default OTel endpoint** is `http://localhost:4318` (OTLP/HTTP). The
   `docker-compose.full.yml` stack includes a collector at this address; the
   slim `docker-compose.yml` does not.
10. **`OTEL_ENABLED=false` (default)** disables export but **does not** skip
    the patching — keep the import order rule even when OTel is off.

## Metrics

11. **Metrics are exposed at `/metrics` on the HTTP port** (3000 by default).
    The standalone Prometheus scrape port `METRICS_PORT=9464` is reserved for
    a future split; today the same metrics surface on both.
12. **Custom gauges live under `src/observability/metrics/`.** The shipped
    `kafka_consumer_lag` gauge is polled every `CONSUMER_LAG_POLL_MS`
    (15s default). Add new gauges in the same directory; register them
    through `MetricsModule`.
13. **Naming**: Prometheus convention is `snake_case` with the unit as a
    suffix (`_seconds`, `_bytes`, `_total` for counters). Don't include the
    `kafka_` prefix unnecessarily — only when the metric is Kafka-specific.

## Health

14. **`/health` covers Kafka + Schema Registry indicators** — see
    `src/health/`. Add a new indicator (rather than expanding an existing one)
    when integrating a new external dependency.
15. **Health checks must time out fast** (default `2s`). A slow upstream
    shouldn't make `/health` unresponsive — k8s readiness probes will start
    flapping.
16. **Don't return 200 for "degraded"** — Kubernetes treats anything in the
    2xx range as healthy. Use `503 Service Unavailable` when an indicator
    fails.

## Correlation IDs

17. **Always propagate `eventId`** through downstream calls triggered by a
    Kafka event. It's the cheapest correlation key — UUID v4 generated by the
    producer (see `UserCreatedProducer.emit`).
18. **Log Orkes `workflowId`** when starting workflows from the app. The
    `OrkesController.startTestWorkflow` already does this; follow the pattern
    if you add new entry points.

## Pre-merge checklist (anything touching observability)

- [ ] No `console.log` introduced; uses `Logger` or pino
- [ ] OTel import order preserved in `src/main.ts`
- [ ] New metrics named per Prometheus convention with units in the suffix
- [ ] New health indicator times out within 2s
- [ ] Required log fields present on any new Kafka handler
- [ ] No raw payloads in INFO+ log lines
